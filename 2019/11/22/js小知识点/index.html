1. void

JavaScript具有一元void运算符。
您可能已经看到它用作void（0）或void 0。
它在生活中有一个单一目的-评估表达式的正确位置并返回未定义的表达式。
习惯使用“ 0”。
您不一定必须使用“ 0”，它可以是任何有效的表达式，例如void <expression>，并且仍返回undefined

Well, turns out, before ES5 you could actually assign a new value to the original undefined likeundefined = "abc" , in most browsers.
Hence, defining the undefined!
In those days, using void was a way to ensure, you are always returning the original undefined.

3. IIFE Brackets can be skipped

  • The syntax for IIFE (Immediately Invoked Functional Expression) was always a bit odd for me. What’s up will all the brackets?

  • 事实证明，只需要这些额外的括号即可告诉JavaScript解析器即将发布的代码是函数表达式，而不是函数。可以想象，知道了这一点，有很多方法可以跳过那些多余的括号，并且仍然可以制作出有效的IIFE。

    // normal IIEF
(function() {
    console.log('Normal IIEF called');
})()

// cool IIEF called
void function() {
    console.log('cool IIEF called');
}()

void运算符告诉解析器代码是函数表达式。
因此，我们可以跳过函数定义的括号
你猜怎么着？我们可以使用任何一元运算符（void，+，！，-等），但这仍然有效！

// IIEF with return

result = (function() {
    // some code
    return "结果"
})()

// IIEF with return

result = function() {
    // some code
    return "结果"
}()

5. The Function constructor

// Function constructor
const multipy = new Function('x', 'y', 'return x + y');
// 最后一个参数是代码  之前都是参数

函数构造函数是JavaScript中所有构造函数的母亲。甚至Object的构造函数都是Function构造函数。
Function自己的构造函数也是Function本身。

因此，调用object.constructor.constructor ...足够的次数最终将返回JavaScript中任何对象上的Function构造函数。

10. Comma operator 逗号运算符

JavaScript具有逗号运算符。它允许我们在一行中编写多个用逗号分隔的表达式，并返回最后一个表达式的结果

used Comma operator in a for loop

for (var a = 0, b = 10; a <= 10; a++, b--)


有时在一行中编写多个语句时会有所帮助

function getNextValue() {
    return counter++, console.log(counter), counter
}


or writing short lamda functions

const getSquare = x => (console.log (x), x * x)

11. +加号运算符

是否曾经想过将字符串快速转换为数字？

只需在字符串前面加上+运算符即可。
加号运算符还适用于负，八进制，十六进制，指数值。
而且，它甚至可以将Date或Moment.js对象转换为时间戳

+'9.11'              // 9.11
+'-4'               // -4
+'0xFF'            // 255
+true             // 1
+'123e-5'        // 0.00123
+false          // 0
+null          // 0
+'Infinity'   // Infinity
+'1,234'     // NaN
+dateobject // timestamp
