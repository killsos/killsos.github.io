什么时候用或不用箭头函数

Posted on 2019-11-22

arrow function

. 省略{}

. 省略 return

  1. No parameters

    () => 42

甚至可以省略括号

_ => 42

  2. single parameter

    x => 42  || (x) => 42

  1. Multiple parameters

    (x, y) => 42

  1. Statements (as opposed to expressions)

    需要花括号

需要return

  2. “Block body”

  3. Object literals

    如果想返回一个对象 需要一个大括号包起来 避免认为是语句

x =>({ y: x })

arrow function Syntactically anonymous

  1. 难于调试

  2. No self-referencing

    递归

event handler that needs to unbind

Main benefit: No binding of ‘this’

var obj = {
    id: 42,
    counter: function counter() {
        setTimeout(() => {
        console.log(this.id);
        }, 1000);
    }
}; // 这是原作者举得栗子

obj.counter() // this --> obj

实际上 arrow functio 中 this 取决于 词法作用域上父级作用域的this 如果父级作用域中this存在

When you should not use Arrow Functions

  1. Object methods

    var cat = {
    lives: 9,
    jumps: () => {
        this.lives--; // this 不是 对象本身
    }
}

  2. Callback functions with dynamic context

    var button = document.getElementById('press');
    button.addEventListener('click', () => {
    this.classList.toggle('on');
}); // this 不是 button

  3. call apply bind 对于 箭头函数 是无效的

  4. 代码可读性差

When you should use them

  1. 箭头函数最适合需要利用this绑定特性，而不是函数本身

  2. map reducer方法中使用 the pros outweigh the cons 利大于弊
